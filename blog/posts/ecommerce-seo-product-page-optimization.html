1. ECサイトSEOの重要性と特徴

ECサイトにとって、SEOは最も費用対効果の高い集客チャネルの一つです。広告と異なり、一度上位表示されれば継続的な流入が見込めます。

📊 ECサイトSEOの重要データ（2025年）

  • 検索経由のECサイト訪問者のコンバージョン率：平均2.8%（広告の1.9%より高い）
  • Google検索の1位クリック率：約28%、2位：約15%、3位：約11%
  • ECサイト訪問者の44%が検索エンジンから流入
  • オーガニック検索からの流入は、広告より平均購入金額が23%高い

出典：BrightEdge, Backlinko, Wolfgang Digital

ECサイトSEOの特徴と課題

ECサイトのSEOは、一般的なコンテンツサイトとは異なる特徴と課題があります：

特徴：

  • 膨大なページ数：数千〜数万の商品ページが存在
  • 重複コンテンツのリスク：似た商品の説明文が重複しやすい
  • 在庫切れ・販売終了：ページの削除・復活が頻繁に発生
  • 構造化データの重要性：価格・在庫・レビューをGoogleに伝える必要がある
  • 画像の重要性：ビジュアルが購買判断に直結

一般的な課題：

  • 商品説明文がメーカーの文章をそのまま流用している（重複コンテンツ）
  • 商品タイトルにキーワードが不足している
  • 画像のファイル名が「IMG_001.jpg」などで最適化されていない
  • 構造化データが実装されていない
  • 表示速度が遅い（大量の商品画像が原因）

本記事では、これらの課題を解決し、商品ページを検索上位表示させるための10の具体的施策を紹介します。

SEO対策の基礎を理解した上で、本記事のECサイト特化施策を実践することで、より効果的です。

2. 商品ページタイトルの最適化

商品ページのtitleタグは、SEOにおいて最も重要な要素の一つです。検索結果に表示されるだけでなく、検索順位に大きく影響します。

効果的な商品タイトルの構成

商品タイトルの基本フォーマット

【ブランド名】商品名 仕様/特徴 ｜ ショップ名

例：

  • 【Nike】エアマックス270 メンズ ランニングシューズ 全5色 ｜ 〇〇スポーツ
  • 【無印良品】ポリプロピレンファイルボックス A4サイズ ホワイト ｜ 〇〇雑貨店
  • 【Canon】EOS R6 Mark II ミラーレス一眼カメラ ボディ ｜ 〇〇カメラ

タイトル最適化の7つのポイント

①キーワードを前方に配置

重要なキーワード（商品名、ブランド名）はできるだけタイトルの前方に配置します。Googleは前方のキーワードをより重視します。

  • ✅ 良い例：「【Nike】エアマックス270 メンズ ランニングシューズ」
  • ❌ 悪い例：「最新モデル入荷！ニューバランス 格安通販 エアマックス270」

②具体的な仕様・特徴を含める

サイズ、色、素材、対応機種など、ユーザーが検索しそうな具体的情報を含めます。

  • 「iPhone 15 Pro ケース クリア 耐衝撃 MagSafe対応」
  • 「ゲーミングチェア オットマン付き リクライニング160度 耐荷重150kg」

③文字数は30〜35文字（全角）が理想

Google検索結果に表示されるのは約30〜35文字（PC）、約40文字（スマホ）。重要な情報がすべて表示される範囲に収めます。

④ブランド名を含める

ブランドで検索するユーザーが多いため、必ず含めます。特に有名ブランドは検索数が多い。

⑤不要な装飾を削除

「【送料無料】」「★人気★」「格安」などの装飾は、SEO効果がほとんどなく、文字数を無駄に消費します。

  • ❌ 悪い例：「★送料無料★【在庫あり】人気No.1！格安！〇〇」
  • ✅ 良い例：「【ブランド名】〇〇 仕様 ｜ ショップ名」

⑥型番・品番を含める（該当する場合）

家電、カメラ、PCパーツなど、型番で検索されることが多い商品は、必ず型番を含めます。

  • 「【ソニー】ワイヤレスヘッドホン WH-1000XM5 ノイズキャンセリング」

⑦競合と差別化する

同じ商品を扱う競合が多い場合、独自の強み（即日発送、保証期間、特典など）をタイトルに含めることも検討。ただし、SEOキーワードを優先。

💡 タイトル最適化のチェックリスト

  • ✅ ブランド名が含まれている
  • ✅ 商品名が正確に記載されている
  • ✅ 主要な仕様（サイズ・色・素材等）が含まれている
  • ✅ 型番が含まれている（該当商品の場合）
  • ✅ 30〜35文字以内に収まっている
  • ✅ 重要キーワードが前方に配置されている
  • ✅ 不要な装飾語が削除されている
  • ✅ 他ページとタイトルが重複していない

キーワード選定方法も併せて参考にしてください。

3. 商品説明文のSEOライティング

商品説明文は、SEOだけでなく購買決定にも影響する重要な要素です。「SEOに強く、かつユーザーが読みたくなる」説明文を作成しましょう。

SEOに強い商品説明文の条件

①オリジナルコンテンツであること

最も重要なのは、メーカーの説明文をコピペしないことです。他のECサイトと同じ説明文は「重複コンテンツ」としてGoogleから評価されません。

重複コンテンツのリスク

  • Googleがオリジナルと判断したページのみが上位表示される
  • あなたのページが後発の場合、ほぼ上位表示されない
  • サイト全体の評価が下がる可能性がある

対策：

  • 自社で撮影した商品写真を使い、実際に使用した感想を記載
  • メーカー説明を参考にしつつ、自分の言葉で書き直す
  • ユーザー目線での使用シーン、メリット・デメリットを追加

②適切な文字数

商品説明文の理想的な文字数は300〜800文字。短すぎるとSEO効果が薄く、長すぎるとユーザーが読まない。

  • シンプルな商品（雑貨、アパレル等）：300〜500文字
  • 複雑な商品（家電、PC、カメラ等）：500〜800文字
  • 高額商品、専門性の高い商品：800〜1,500文字

③検索キーワードを自然に含める

タイトルに含めたキーワードを、説明文にも自然な形で2〜4回程度含めます。ただし、詰め込みすぎは逆効果。

  • ✅ 良い例：「このゲーミングチェアは、長時間のゲームプレイでも疲れにくい設計です。リクライニング機能は最大160度まで倒れ、オットマンも搭載しているため...」
  • ❌ 悪い例：「ゲーミングチェア ゲーミングチェア ゲーミングチェアを探しているなら当店のゲーミングチェアがおすすめ...」

効果的な商品説明文の構成

商品説明文の理想的な構成

  1. リード文（50〜100文字）：商品の魅力を一言で
    • 例：「2025年最新モデルのノイズキャンセリングヘッドホン。業界最高クラスの静寂性で、通勤・在宅ワークを快適に。」
  2. 主要な特徴・メリット（150〜300文字）：箇条書きまたは段落で
    • 例：「本製品の3つの特徴：①ノイズキャンセリング性能が前モデル比120%向上 ②最大30時間連続再生可能 ③軽量設計で長時間装着しても疲れない」
  3. 使用シーン・こんな方におすすめ（100〜200文字）
    • 例：「通勤・通学での音楽鑑賞、在宅ワークでの集中環境作り、オンライン会議でのクリアな音声通話に最適です。音質にこだわる方、長時間使用される方に特におすすめ。」
  4. 詳細スペック・仕様：箇条書きまたは表形式で
    • 重量、サイズ、対応機種、素材、保証期間など

読みやすさを向上させるテクニック

①段落を短くする

スマホで見たときに、1段落が画面いっぱいにならないよう、2〜3行で改行。

②箇条書きを活用する

特徴やメリットは箇条書き（ul/olタグ）で見やすく。

③見出しを使う

長い説明文の場合、h3やh4タグで見出しを入れると読みやすい。

④表を活用する

スペック情報は表（tableタグ）で整理すると、ユーザーにもGoogleにも分かりやすい。

📝 商品説明文チェックリスト

  • ✅ オリジナルの文章になっている（メーカーコピペでない）
  • ✅ 300文字以上ある
  • ✅ 主要キーワードが2〜4回含まれている（自然な形で）
  • ✅ 段落が短く、読みやすい
  • ✅ 箇条書き・表を活用している
  • ✅ 使用シーン・メリットが明確
  • ✅ スペック情報が網羅されている
  • ✅ 誤字脱字がない

SEOライティングの基礎も参考にしてください。

4. 構造化データ（Product Schema）の実装

構造化データ（Structured Data）を実装することで、Googleに商品情報を正確に伝え、検索結果にリッチスニペット（価格、在庫、レビュー等）を表示できます。

📊 構造化データの効果

  • 検索結果にレビュー星評価が表示され、CTRが30〜40%向上
  • 価格・在庫状況が表示され、ユーザーの購買意欲を刺激
  • Googleショッピングタブに掲載される可能性が高まる
  • 音声検索での回答に採用されやすくなる

商品ページに必須の構造化データ

ECサイトの商品ページには、Product Schema（商品スキーマ）を実装します。

Product Schemaの実装例（JSON-LD形式）

<script type="application/ld+json">
{
  "@context": "https://schema.org/",
  "@type": "Product",
  "name": "【Nike】エアマックス270 メンズ ランニングシューズ",
  "image": [
    "https://example.com/photos/airmax270-1.jpg",
    "https://example.com/photos/airmax270-2.jpg"
  ],
  "description": "2025年最新モデルのエアマックス270。軽量で通気性に優れ...",
  "sku": "AM270-BLK-27",
  "mpn": "AM270",
  "brand": {
    "@type": "Brand",
    "name": "Nike"
  },
  "offers": {
    "@type": "Offer",
    "url": "https://example.com/products/airmax270",
    "priceCurrency": "JPY",
    "price": "15800",
    "priceValidUntil": "2025-12-31",
    "availability": "https://schema.org/InStock",
    "seller": {
      "@type": "Organization",
      "name": "〇〇スポーツ"
    }
  },
  "aggregateRating": {
    "@type": "AggregateRating",
    "ratingValue": "4.7",
    "reviewCount": "328"
  },
  "review": {
    "@type": "Review",
    "reviewRating": {
      "@type": "Rating",
      "ratingValue": "5"
    },
    "author": {
      "@type": "Person",
      "name": "山田太郎"
    },
    "reviewBody": "軽くて履き心地が最高です！デザインもかっこいい。"
  }
}
</script>

必須プロパティと推奨プロパティ

必須プロパティ：

  • name：商品名
  • image：商品画像URL（最低1枚、複数推奨）
  • offers：価格・在庫情報
    • price：価格（税込）
    • priceCurrency：通貨（JPY）
    • availability：在庫状況（InStock / OutOfStock / PreOrder）

推奨プロパティ（実装することで効果向上）：

  • description：商品説明
  • brand：ブランド名
  • sku：商品管理番号
  • mpn：メーカー型番
  • aggregateRating：平均評価（星評価）と総レビュー数
  • review：個別レビュー

在庫切れ商品の扱い

在庫切れ商品でも、ページを残す場合は構造化データも残します。availabilityをOutOfStockに変更。

"availability": "https://schema.org/OutOfStock"

再入荷予定がある場合はPreOrderを使用：

"availability": "https://schema.org/PreOrder"

構造化データのテストと検証

実装後、必ず以下のツールで検証します：

  1. リッチリザルトテスト：https://search.google.com/test/rich-results
    • 構造化データが正しく認識されているか確認
  2. Google Search Console：「拡張」セクション
    • 実際のインデックス状況とエラーを確認

💡 構造化データ実装のポイント

  • JSON-LD形式が最も推奨される（HTMLに直接書く必要がない）
  • 価格は税込価格を記載
  • 画像は高解像度（最低1200px幅）を使用
  • レビュー評価は実際のユーザーレビューのみ（自作はNG）
  • 在庫状況は常に最新に保つ
  • バリエーション商品（色・サイズ違い）は個別に構造化データを実装

構造化データ実装の詳細ガイドも併せてご覧ください。

5. 画像最適化とAlt属性

ECサイトでは、商品画像が購買決定に直結するため、高品質な画像が必須です。同時に、SEO対策としての画像最適化も重要です。

画像SEOの3つの柱

①ファイル名の最適化

画像のファイル名は、Googleが画像内容を理解する重要な手がかりです。

  • ❌ 悪い例：IMG_20250119_001.jpg、product-1.jpg、image.jpg
  • ✅ 良い例：nike-airmax270-black-mens-running-shoes-front.jpg

ファイル名のルール：

  • 商品名・ブランド名を含める
  • 主要な特徴（色、サイズ等）を含める
  • 英数字とハイフン(-)のみ使用（日本語は避ける）
  • 複数画像の場合は、アングル名を追加（front, side, back, detail等）

②Alt属性（代替テキスト）の最適化

Alt属性は、画像が表示できない場合に代わりに表示されるテキストで、SEOにおいて非常に重要です。

<img src="nike-airmax270-black-mens.jpg" alt="ナイキ エアマックス270 メンズ ランニングシューズ ブラック 正面">

Alt属性のルール：

  • 画像の内容を正確に説明（50〜125文字が理想）
  • 商品名・ブランド名を含める
  • 色・サイズ・アングルなど具体的情報を含める
  • 「画像」「写真」などの不要な言葉は入れない
  • キーワードの詰め込みはしない
  • ❌ 悪い例：「商品画像」「靴」「エアマックス エアマックス エアマックス」
  • ✅ 良い例：「ナイキ エアマックス270 メンズ ランニングシューズ ブラック 正面から見た画像」

③画像ファイルサイズの最適化

大きすぎる画像ファイルは、ページ表示速度を遅くし、SEOに悪影響を与えます。

画像最適化の目標値

  • メイン商品画像：100KB以下（最大200KB）
  • サムネイル画像：20〜50KB
  • 詳細・拡大画像：200〜300KB

推奨フォーマット：

  • WebP：最も軽量で高品質（2025年現在、主要ブラウザすべてで対応）
  • JPEG：写真向け（品質80〜90%で圧縮）
  • PNG：透過が必要な場合のみ

画像圧縮ツール：

  • TinyPNG：https://tinypng.com/（無料、品質を保ちつつ圧縮）
  • Squoosh：Google製の画像最適化ツール
  • ImageOptim：Mac向けアプリ
  • ShortPixel：WordPressプラグイン

画像の解像度と表示サイズ

推奨解像度：

  • メイン商品画像：1200px × 1200px以上（拡大表示に対応）
  • サムネイル：300px × 300px
  • 詳細画像：2000px × 2000px（ズーム機能用）

アスペクト比：

  • ECサイトでは1:1（正方形）が最も一般的
  • 全商品で統一することで、見た目が整う

遅延読み込み（Lazy Loading）の実装

ページ表示速度を改善するため、スクロールして表示されるまで画像を読み込まない「遅延読み込み」を実装します。

<img src="product.jpg" alt="商品名" loading="lazy">

ただし、ファーストビュー（スクロールせずに見える部分）の画像には使用しない。メイン商品画像は即座に表示されるべき。

📸 画像SEOチェックリスト

  • ✅ ファイル名が商品情報を含む適切な名前になっている
  • ✅ すべての画像にAlt属性が設定されている
  • ✅ Alt属性が画像内容を正確に説明している
  • ✅ 画像がWebPまたは最適化されたJPEGで保存されている
  • ✅ メイン画像が100KB以下に圧縮されている
  • ✅ 画像解像度が1200px以上ある
  • ✅ 遅延読み込み（loading="lazy"）が実装されている（ファーストビュー以外）
  • ✅ 複数アングルの商品画像がある（最低3〜5枚）

6. URLスラッグ設計

商品ページのURLは、SEOと ユーザビリティの両方に影響します。適切なURL設計を行いましょう。

SEOに強いURLの原則

①シンプルで短いURL

  • ✅ 良い例：https://example.com/products/nike-airmax270-black
  • ❌ 悪い例：https://example.com/p?id=12345&cat=shoes&color=black

②キーワードを含める

URLにブランド名・商品名を含めることで、SEO効果が高まります。

  • https://example.com/products/canon-eos-r6-mark2
  • https://example.com/products/iphone15-pro-case-clear

③英数字とハイフンのみ使用

日本語URLは避け、英語表記にします。日本語URLはエンコードされると非常に長くなり、共有しづらくなります。

  • ❌ 悪い例：https://example.com/products/エアマックス270-黒
    • → エンコード後：https://example.com/products/%E3%82%A8%E3%82%A2...
  • ✅ 良い例：https://example.com/products/airmax270-black

④階層構造を明確にする

URLの階層で、カテゴリ構造を表現します。

  • https://example.com/products/nike-airmax270
  • https://example.com/mens/shoes/nike-airmax270
  • https://example.com/category/running-shoes/nike-airmax270

ただし、階層が深すぎる（4階層以上）のは避けましょう。

カテゴリページのURL設計

商品ページだけでなく、カテゴリページのURLも重要です。

  • https://example.com/category/mens-shoes
  • https://example.com/category/running-shoes
  • https://example.com/brand/nike

URLの変更とリダイレクト

すでに公開されている商品ページのURLを変更する場合、必ず301リダイレクトを設定します。リダイレクトしないと、これまで蓄積したSEO評価が失われます。

301リダイレクトの設定例（.htaccess）

Redirect 301 /old-product-url /new-product-url

🔗 URLスラッグ設計のベストプラクティス

  • ブランド名-商品名-色/サイズ の順で構成
  • 単語の区切りはハイフン(-)を使用（アンダースコア_はNG）
  • 小文字のみ使用（大文字と小文字を混在させない）
  • 不要な単語（a, the, and等）は削除
  • 型番がある場合は含める
  • 一度公開したURLは変更しない（変更する場合は301リダイレクト必須）

7. 内部リンク戦略

ECサイトの内部リンク構造を最適化することで、SEO効果とユーザー体験の両方を向上させることができます。

ECサイトにおける内部リンクの重要性

  • クロール効率の向上：Googleのクローラーがサイト全体を巡回しやすくなる
  • ページランクの分配：重要なページにリンクを集めることで、SEO評価を高める
  • ユーザー回遊率の向上：関連商品を提案し、購買機会を増やす
  • コンバージョン率の向上：適切な導線設計で購入までスムーズに誘導

商品ページに設置すべき内部リンク

①パンくずリスト（必須）

パンくずリストは、ユーザーとGoogleの両方にサイト構造を伝える重要な要素です。

ホーム > メンズシューズ > ランニングシューズ > ナイキ エアマックス270

②関連商品・類似商品リンク

  • 「この商品を見た人はこちらも見ています」
  • 「同じカテゴリの商品」
  • 「同じブランドの商品」

推奨数：4〜8商品

③カテゴリページへのリンク

  • 商品が属するカテゴリページへのリンク
  • ブランドページへのリンク

④セット商品・コーディネート提案

  • 「この商品と一緒に購入されています」
  • 「おすすめのコーディネート」

⑤サイズ・色違いへのリンク

同一商品の別バリエーションへのリンク。ただし、canonicalタグの設定に注意。

カテゴリページの内部リンク最適化

カテゴリページは、ECサイトのSEOにおいて最も重要なページの一つです。

①階層構造の明確化

  • 親カテゴリ → 子カテゴリ → 商品 の3階層が理想
  • カテゴリページ同士のリンクも設置（サイドバー等）

②商品一覧での内部リンク

  • 商品画像とタイトルの両方にリンク設置
  • 1ページあたり20〜50商品が理想（多すぎると表示速度が遅くなる）

③ファセット（絞り込み）の扱い

色・サイズ・価格帯などで絞り込んだページは、重複コンテンツになりやすいため注意。

  • 絞り込みページはnoindexまたはcanonicalで元のカテゴリページを指定
  • またはrobots.txtでクロールをブロック

アンカーテキストの最適化

内部リンクのアンカーテキスト（リンクテキスト）は、リンク先ページのSEOに影響します。

  • ❌ 悪い例：「こちら」「詳細はこちら」「クリック」
  • ✅ 良い例：「ナイキ エアマックス270 商品ページ」「メンズランニングシューズ一覧」

リンク先のページ内容を具体的に示すテキストを使用します。

🔗 内部リンク最適化のポイント

  • すべての商品ページにパンくずリストを設置
  • 関連商品リンクは4〜8商品が理想
  • アンカーテキストは具体的に（「こちら」はNG）
  • 孤立ページ（どこからもリンクされていないページ）をなくす
  • 重要なページ（売れ筋商品、利益率の高い商品）により多くのリンクを集める
  • リンク切れを定期的にチェック（Screaming Frog等のツール使用）

SEO対策の基礎で内部リンクについても詳しく解説しています。

8. レビュー・評価の活用

ユーザーレビューと評価は、ECサイトのSEOとコンバージョン率の両方に大きく影響します。

レビューがSEOに与える効果

⭐ レビューのSEO効果

  • 検索結果に星評価が表示：CTRが20〜35%向上
  • ユニークコンテンツの増加：レビュー文章がオリジナルコンテンツとして評価される
  • ロングテールキーワードの獲得：ユーザーが様々な言葉で商品を表現
  • フレッシュネス：新しいレビューが定期的に追加されることで、ページが更新される
  • 信頼性の向上：Googleは「E-E-A-T（経験・専門性・権威性・信頼性）」を重視

レビュー機能の実装

①レビュー投稿フォームの設置

  • 購入後、一定期間経過（1〜2週間）後にレビュー依頼メールを送信
  • 簡単に投稿できるフォーム（星評価+コメント）
  • 画像投稿機能（あると信頼性向上）

②レビューの表示

  • 商品ページに平均評価と総レビュー数を目立つ位置に表示
  • 個別レビューを5〜10件表示（全件は別ページまたはタブで表示）
  • レビューの並び順：「参考になった」順、新着順、評価順など選択可能に

③レビューの構造化データ実装

前述のProduct Schemaに、aggregateRatingとreviewを必ず含めます。

レビューを増やす施策

①レビュー投稿のインセンティブ

  • レビュー投稿でポイント付与（100〜500ポイント）
  • 次回使えるクーポン配布
  • 抽選でプレゼント

ただし、良いレビューを書くことを条件にしてはいけません（ステマになる）。「レビューを書いたら」という条件のみOK。

②購入後フォローメール

  • 購入後1〜2週間後に「商品はいかがでしたか？」メールを送信
  • メール内にレビュー投稿リンクを設置

③レビュー投稿の簡略化

  • ログイン不要で投稿できる（ただし承認制にしてスパム対策）
  • スマホから簡単に投稿できるUI
  • テンプレート提供（「サイズ感」「品質」「デザイン」を選択式で評価）

ネガティブレビューの対応

低評価レビューは削除したくなりますが、以下の理由で残すべきです：

  • 平均評価4.2〜4.7が最も信頼される（5.0満点は逆に疑われる）
  • ネガティブレビューがあることで、ポジティブレビューの信頼性が高まる
  • ネガティブレビューへの誠実な対応で、企業の姿勢をアピールできる

ネガティブレビューへの対応：

  • 24時間以内に返信する
  • 謝罪と改善策を明示
  • 個別対応を申し出る（メール・電話での対応）

削除すべきレビュー：

  • 商品と無関係の内容
  • 誹謗中傷・攻撃的な表現
  • 競合他社の宣伝
  • 明らかなスパム

⭐ レビュー活用のポイント

  • 最低でも5件以上のレビューがある状態を目指す
  • 平均評価は4.2〜4.7が理想（高すぎても疑われる）
  • レビューの構造化データを必ず実装
  • 購入後フォローメールで能動的にレビュー依頼
  • ネガティブレビューも残し、誠実に対応
  • 画像付きレビューは特に価値が高い
  • レビューへの返信で企業の姿勢をアピール

9. サイト表示速度改善

ページ表示速度は、2025年現在、Googleの重要なランキング要因の一つです。特にECサイトは画像が多いため、表示速度が遅くなりがちです。

⚡ 表示速度の重要性

  • ページ読み込みが1秒遅くなるごとに、コンバージョン率が7%低下
  • 表示速度が3秒以上かかると、53%のユーザーが離脱
  • GoogleのCore Web Vitalsが検索順位に直接影響
  • モバイルユーザーは特に表示速度に敏感

Core Web Vitalsとは

Googleが重視する3つのユーザー体験指標：

①LCP（Largest Contentful Paint）

  • 最も大きなコンテンツが表示されるまでの時間
  • 目標：2.5秒以内
  • 主な改善策：画像最適化、サーバー応答速度改善

②FID（First Input Delay）/ INP（Interaction to Next Paint）

  • ユーザーが最初に操作してから反応するまでの時間
  • 目標（FID）：100ミリ秒以内
  • 目標（INP）：200ミリ秒以内
  • 主な改善策：JavaScript最適化

③CLS（Cumulative Layout Shift）

  • ページ読み込み中のレイアウトのずれ
  • 目標：0.1以下
  • 主な改善策：画像・動画にサイズ指定、フォント最適化

ECサイト表示速度改善の10施策

①画像最適化（最重要）

  • WebP形式を使用
  • 適切なサイズに圧縮（100KB以下目標）
  • Lazy Loading（遅延読み込み）実装
  • レスポンシブ画像（srcset属性）使用

②CDN（コンテンツデリバリーネットワーク）の利用

  • 画像をユーザーに近いサーバーから配信
  • 推奨CDN：Cloudflare、Amazon CloudFront、Fastly

③ブラウザキャッシュの活用

  • 一度読み込んだ画像・CSSをブラウザに保存
  • 再訪問時の読み込み時間を大幅短縮

④CSS・JavaScriptの最小化

  • 不要な空白・改行を削除（Minify）
  • 複数ファイルを1つに結合

⑤不要なJavaScriptの削除

  • 使っていないプラグインを削除
  • 外部スクリプト（SNSウィジェット等）を最小限に

⑥HTMLの最適化

  • 不要なタグやコメントを削除
  • インラインCSSは最小限に

⑦サーバー応答速度の改善

  • 高速なホスティングサービスを利用
  • サーバーサイドキャッシュ実装
  • データベースクエリの最適化

⑧フォントの最適化

  • Webフォントは必要最小限に
  • font-display: swap で先にテキストを表示
  • 使用する文字のみをサブセット化

⑨AMPの実装（オプション）

  • Accelerated Mobile Pages（超高速モバイルページ）
  • 特にモバイルユーザーが多いサイトで有効

⑩定期的な速度測定

  • PageSpeed Insights：https://pagespeed.web.dev/
  • GTmetrix：詳細な分析レポート
  • Google Search Console：Core Web Vitalsレポート

⚡ 表示速度改善の優先順位

  1. 画像最適化：最も効果が大きい
  2. 不要なJavaScript削除：特にサードパーティスクリプト
  3. CDN導入：グローバル展開している場合は必須
  4. ブラウザキャッシュ：設定するだけで効果大
  5. サーバー高速化：根本的な改善

10. モバイルSEO対策

2025年現在、ECサイトのトラフィックの70%以上がモバイルからです。Googleも「モバイルファーストインデックス」を採用しており、モバイル版ページが検索順位の基準になります。

モバイルファーストインデックスとは

Googleは2018年から、デスクトップ版ではなくモバイル版ページを評価基準にしています。つまり、モバイルで見にくいサイトは、デスクトップ検索でも順位が下がります。

モバイルSEOの必須対策

①レスポンシブデザイン

画面サイズに応じてレイアウトが自動調整されるデザイン。最も推奨される方法。

<meta name="viewport" content="width=device-width, initial-scale=1.0">

すべてのページに必ず設置。

②タップ可能な要素のサイズ

  • ボタン・リンクは最低48px × 48px以上
  • ボタン間の間隔は最低8px以上
  • 小さすぎると誤タップが発生し、ユーザビリティ低下

③読みやすいフォントサイズ

  • 本文テキストは最低16px以上
  • 拡大しなくても読める大きさ

④横スクロールを避ける

  • コンテンツが画面幅に収まるようにする
  • 画像・表が画面からはみ出さないように

⑤ポップアップ・インタースティシャル広告の制限

Googleは、モバイルで全画面を覆うポップアップを嫌います。特に、ページ読み込み直後に表示されるポップアップはペナルティの対象。

  • ❌NG：ページ表示直後に「メルマガ登録」の全画面ポップアップ
  • ✅OK：ページ下部に控えめなバナー、または一定時間経過後に表示

⑥モバイル専用の購入導線

  • 「カートに入れる」ボタンは常に画面下部に固定表示（Sticky）
  • 入力フォームは最小限に（住所自動入力、クレカ番号のコピペ対応等）
  • チェックアウトプロセスは3ステップ以内

モバイルページ速度の最適化

モバイルはデスクトップより回線速度が遅いため、表示速度がより重要。

  • モバイル版PageSpeed Insightsでスコア90以上を目指す
  • 画像は特に軽量化（50KB以下推奨）
  • AMP実装を検討

モバイルユーザビリティのテスト

①Google Search Console

  • 「モバイルユーザビリティ」レポートでエラーを確認
  • 「テキストが小さすぎる」「タップ要素が近すぎる」等の警告に対処

②実機テスト

  • iPhone・Android実機で実際に操作してみる
  • 購入フローを最初から最後まで完了できるかテスト

③ヒートマップツール

  • Microsoft Clarity（無料）
  • Hotjar

モバイルユーザーがどこをタップしているか、どこで離脱しているかを分析。

📱 モバイルSEOチェックリスト

  • ✅ レスポンシブデザインが実装されている
  • ✅ viewport metaタグが設置されている
  • ✅ ボタン・リンクが48px×48px以上ある
  • ✅ フォントサイズが16px以上ある
  • ✅ 横スクロールが発生しない
  • ✅ ポップアップが控えめ（または遅延表示）
  • ✅ モバイル版PageSpeed Insightsスコア80以上
  • ✅ Google Search Consoleでモバイルユーザビリティエラーがない
  • ✅ 実機でスムーズに購入完了できる

よくある質問（FAQ）

Q1. ECサイトのSEO対策で最も重要なことは何ですか？

最も重要なのは商品ページタイトルの最適化と構造化データの実装です。この2つだけでも、検索結果での表示が大きく改善し、CTRとトラフィックが向上します。次に、オリジナルの商品説明文を書き、画像を最適化することで、さらに効果が高まります。

Q2. メーカーの商品説明をそのまま使っても大丈夫ですか？

NGです。メーカーの説明文をそのまま使うと「重複コンテンツ」となり、Googleから評価されません。他のECサイトも同じ説明文を使っている場合、あなたのページは検索結果に表示されない可能性が高いです。必ず自分の言葉で書き直し、実際に使った感想や使用シーンを追加しましょう。

Q3. 構造化データは必須ですか？実装は難しいですか？

必須ではありませんが、実装することで大きなSEO効果が得られます。構造化データを実装すると、検索結果に価格・在庫・レビュー星評価が表示され、CTRが30〜40%向上します。実装はJSON-LD形式であれば、HTMLに1つスクリプトタグを追加するだけなので、それほど難しくありません。Shopifyなどのプラットフォームを使っている場合は、自動で実装されることもあります。

Q4. 在庫切れ商品のページは削除すべきですか？

削除しないことを推奨します。理由：①これまで蓄積したSEO評価が失われる、②再入荷した際に再度SEO評価を築く必要がある、③「在庫切れ」として残しておくことで、再入荷通知登録などのリード獲得ができる。

対策として、構造化データのavailabilityをOutOfStockに変更し、ページに「在庫切れ」と明記します。類似商品へのリンクや再入荷通知登録フォームを設置すると、ユーザー体験も向上します。

Q5. 画像のファイル名やAlt属性は本当にSEOに影響しますか？

はい、影響します。特にGoogleの画像検索経由の流入を狙う場合は重要です。適切なファイル名とAlt属性を設定することで、画像検索での上位表示が可能になり、追加のトラフィックを獲得できます。ECサイトでは、画像検索からの流入が全体の5〜15%を占めることもあるため、無視できません。

Q6. ECサイトのSEO効果が出るまでどのくらいかかりますか？

一般的に3〜6ヶ月かかります。ただし、以下の要因で変動します：

  • 競合の強さ：ニッチなキーワードなら1〜2ヶ月、競合が強いキーワードなら6ヶ月〜1年以上
  • サイトの規模：新規サイトより、既存サイトの方が早い
  • ドメインオーソリティ：被リンクが多いサイトは早く上位表示される

早期に効果を出すには、ロングテールキーワード（「ブランド名 商品名 色」など具体的なキーワード）から狙うと良いでしょう。

ECサイトのSEO対策をプロに任せませんか？

余日（ヨジツ）では、ECサイトに特化したSEO対策サービスを提供しています。商品ページ最適化から、構造化データ実装、表示速度改善まで、トータルでサポートいたします。

余日のECサイトSEO支援サービス

  • 商品ページ最適化：タイトル・説明文・画像・構造化データを一括最適化
  • テクニカルSEO：表示速度改善、モバイル対応、内部リンク最適化
  • コンテンツSEO：カテゴリページ、ブログ記事でオーガニック流入を増加
  • 効果測定とレポート：Google AnalyticsとSearch Consoleで成果を可視化
  • 継続的な改善：定期的な順位チェックとアルゴリズム変動対応

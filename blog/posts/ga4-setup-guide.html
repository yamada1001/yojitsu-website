GA4とは？UAとの違い

GA4（Googleアナリティクス4）は、Googleが提供する最新のアクセス解析ツールです。従来のユニバーサルアナリティクス（UA）とは根本的に設計思想が異なります。

GA4の主な特徴

  • イベントベースの計測：全てのユーザー行動を「イベント」として計測
  • クロスプラットフォーム対応：Web・アプリを統合して計測可能
  • 機械学習の活用：予測指標やインサイト機能
  • プライバシー重視：Cookie規制に対応した設計
  • BigQuery連携：無料で生データをエクスポート可能

UAとGA4の主な違い

項目 UA（ユニバーサルアナリティクス） GA4
計測モデル セッションベース イベントベース
データ構造 ページビュー中心 全てイベント
レポート 定型レポート中心 カスタマイズ性が高い
ユーザー識別 主にCookie 複数の識別方法（Cookie + User ID等）
BigQuery連携 有料（360のみ） 無料で利用可能
データ保持期間 無期限（一部制限あり） 最大14ヶ月（設定で変更可）

BtoBマーケティングでGA4を活用する場合、特にイベント計測とコンバージョン設定が重要になります。従来のUAとは考え方が異なるため、しっかりと理解して設定しましょう。

GA4移行の必要性

UAは2023年7月1日で計測を終了しました。既にGA4への移行は「検討事項」ではなく「必須事項」です。まだ移行していない場合は、本記事の手順に従って早急に設定を完了させましょう。

GA4の初期設定手順

GA4の初期設定は、大きく分けて4つのステップで完了します。順を追って見ていきましょう。

ステップ1：GA4プロパティの作成

1. Googleアナリティクスにログイン

Googleアナリティクスにアクセスし、Googleアカウントでログインします。

2. 新しいプロパティを作成

  1. 左下の「管理」（歯車アイコン）をクリック
  2. 「プロパティ」列の「プロパティを作成」をクリック
  3. プロパティ名を入力（例：「会社名 - GA4」）
  4. レポートのタイムゾーンを「日本」に設定
  5. 通貨を「日本円（JPY）」に設定
  6. 「次へ」をクリック

3. ビジネス情報の入力

  1. 業種を選択
  2. ビジネスの規模を選択
  3. Googleアナリティクスの利用目的を選択
  4. 「作成」をクリック
  5. 利用規約に同意

ステップ2：データストリームの設定

1. プラットフォームを選択

Webサイトの場合は「ウェブ」を選択します。

2. ストリームの詳細を設定

  1. ウェブサイトのURLを入力（例：https://example.com）
  2. ストリーム名を入力（例：「メインサイト」）
  3. 「ストリームを作成」をクリック

3. 測定IDを確認

作成されたストリーム画面に「G-XXXXXXXXXX」形式の測定IDが表示されます。このIDを使ってタグを設置します。

ステップ3：タグの設置

GA4のタグを設置する方法は主に2つあります：

方法1：Googleタグを直接HTMLに設置

ストリーム画面の「タグ設定手順」から「新しいページ上のタグを追加する」→「手動でインストール」を選択すると、以下のようなコードが表示されます： 

<!-- Google tag (gtag.js) -->
<script async src="https://www.googletagmanager.com/gtag/js?id=G-XXXXXXXXXX"></script>
<script>
  window.dataLayer = window.dataLayer || [];
  function gtag(){dataLayer.push(arguments);}
  gtag('js', new Date());

  gtag('config', 'G-XXXXXXXXXX');
</script>

このコードを全ページの<head>タグ内に設置します。

方法2：Googleタグマネージャー（GTM）を使用【推奨】

GTMを使用すると、タグ管理が格段に楽になります。詳しくは後述の「GTMとの連携設定」セクションで解説します。

推奨：GTMでの管理

実務では、GTM（Googleタグマネージャー）での管理を強く推奨します。タグの追加・変更・削除が簡単になり、エンジニアに依頼せずマーケターだけで対応できるようになります。リード獲得施策のPDCAを高速で回すためにも必須のツールです。

ステップ4：動作確認

1. リアルタイムレポートで確認

  1. GA4管理画面の左メニューから「リアルタイム」を選択
  2. 自分のサイトにアクセス
  3. リアルタイムレポートに訪問が表示されるか確認

2. DebugViewで詳細確認

より詳細に確認するには、GA4の「DebugView」機能が便利です：

  1. Chrome拡張機能「Google Analytics Debugger」をインストール
  2. 拡張機能を有効化してサイトにアクセス
  3. GA4管理画面の「DebugView」でイベントの発火を確認

データストリームの詳細設定

データストリームには、重要な拡張設定がいくつかあります。必ず設定しておきたい項目を解説します。

拡張計測機能の有効化

データストリーム画面の「拡張計測機能」をONにすることで、以下のイベントが自動的に計測されます：

  • ページビュー：ページの閲覧（page_view）
  • スクロール：ページの90%までスクロール（scroll）
  • 離脱クリック：外部リンクのクリック（click）
  • サイト内検索：検索機能の使用（view_search_results）
  • 動画エンゲージメント：埋め込み動画の再生（video_start, video_progressなど）
  • ファイルのダウンロード：PDFなどのダウンロード（file_download）

これらは基本的に全て有効にしておくことを推奨します。特にリード獲得施策では、ホワイトペーパーのダウンロードなどを自動で計測できるため非常に便利です。

Googleシグナルの有効化

Googleシグナルを有効にすると、以下の機能が利用できます：

  • クロスデバイスのユーザー追跡
  • リマーケティング機能
  • 広告レポート機能
  • ユーザー属性レポート（年齢・性別など）

設定方法

  1. 管理画面から「データ設定」→「データ収集」を選択
  2. 「Googleシグナルのデータ収集」で「開始する」をクリック
  3. 内容を確認して「続行」→「有効にする」をクリック

プライバシーポリシーの更新を忘れずに

Googleシグナルを有効にする場合、サイトのプライバシーポリシーに「Googleアナリティクスによる広告機能を使用している」旨を記載する必要があります。

データ保持期間の設定

GA4のデフォルトのデータ保持期間は2ヶ月です。これを14ヶ月に変更しておきましょう：

  1. 管理画面から「データ設定」→「データ保持」を選択
  2. 「イベントデータの保持」を「14か月」に変更
  3. 「保存」をクリック

ただし、これは探索レポートなどで使えるユーザー単位のデータ保持期間です。標準レポートは無期限で保持されますので安心してください。

イベント計測の設定

GA4の真髄はイベント計測にあります。ビジネスに重要なユーザー行動を全てイベントとして計測しましょう。

イベントの種類

GA4のイベントは4つのタイプに分類されます：

1. 自動収集イベント

GA4タグを設置するだけで自動的に計測されるイベント：

  • first_visit：初回訪問
  • session_start：セッション開始
  • user_engagement：ユーザーエンゲージメント

2. 拡張計測機能イベント

前述の拡張計測機能で自動計測されるイベント（page_view, scroll, clickなど）

3. 推奨イベント

Googleが推奨する標準イベント名：

  • login：ログイン
  • sign_up：会員登録
  • purchase：購入
  • add_to_cart：カート追加
  • generate_lead：リード獲得

4. カスタムイベント

ビジネス独自のイベント（自由に命名可能）

推奨イベントを使うメリット

推奨イベント名を使用すると、GA4の標準レポートに自動的に反映されます。例えばgenerate_leadイベントを使えば、リード獲得数が標準レポートで可視化されます。BtoBマーケティングでは特に重要です。

カスタムイベントの作成方法

方法1：GA4管理画面から作成

既存のイベントを条件に応じて変換できます：

  1. 管理画面から「イベント」を選択
  2. 「イベントを作成」をクリック
  3. イベント名と条件を設定

例：特定のページ閲覧をイベント化

  • イベント名：contact_page_view
  • 条件：page_location 含む /contact/

方法2：GTMから送信【推奨】

より柔軟なイベント計測にはGTMの使用を推奨します。詳細は「GTMとの連携設定」で解説します。

BtoB向け推奨イベント設定例

BtoBマーケティングで計測すべき主要なイベントをご紹介します：

イベント名 計測対象 重要度
generate_lead お問い合わせフォーム送信 ★★★
file_download 資料・ホワイトペーパーDL ★★★
sign_up 無料トライアル申込 ★★★
view_item サービス詳細ページ閲覧 ★★☆
video_complete 紹介動画の視聴完了 ★★☆
phone_click 電話番号のクリック ★☆☆

これらのイベントを適切に設定することで、リード獲得のボトルネックを可視化し、改善につなげられます。

コンバージョン設定

イベントを設定したら、次はコンバージョン（CV）として登録します。これにより、目標達成の計測とレポート作成が可能になります。

コンバージョンの設定手順

1. コンバージョンとして登録

  1. 管理画面から「イベント」を選択
  2. コンバージョンにしたいイベントの「コンバージョンとしてマークを付ける」をONに

または、「管理」→「コンバージョン」→「新しいコンバージョンイベント」から直接設定することも可能です。

2. コンバージョンの確認

設定後、以下のレポートでコンバージョンを確認できます：

  • 「レポート」→「収益化」→「コンバージョン」
  • 「探索」レポートでカスタム分析

BtoB企業の推奨コンバージョン設定

BtoBマーケティングでは、以下のコンバージョンを設定することを推奨します：

マイクロコンバージョン

  • 資料ダウンロード
  • メルマガ登録
  • 無料トライアル申込
  • セミナー申込
  • 料金ページ閲覧

マクロコンバージョン

  • お問い合わせ送信
  • デモ申込
  • 見積もり依頼

マイクロコンバージョンとマクロコンバージョンを両方設定することで、カスタマージャーニー全体を可視化できます。

コンバージョン値の設定

各コンバージョンに金額的な価値を設定することも可能です：

  1. GTMでイベント送信時にvalueパラメータを追加
  2. または、GA4の「イベントの変更」機能でvalueを設定

例えば、お問い合わせ1件あたりの平均受注額が100万円で、成約率が10%なら、お問い合わせのコンバージョン値は10万円（100万円 × 10%）と設定できます。

コンバージョン値設定のメリット

コンバージョン値を設定すると、LP制作や広告運用のROI（投資対効果）が数値で可視化されます。どの施策が最も収益に貢献しているかが一目瞭然になります。

カスタムディメンション・指標の設定

GA4では、標準で用意されていないデータを「カスタムディメンション」や「カスタム指標」として追加できます。

カスタムディメンションとは

ユーザーやイベントに紐づく属性情報です。例：

  • 会員ステータス（無料会員/有料会員）
  • ログイン状態（ログイン済み/未ログイン）
  • 記事カテゴリ
  • フォーム種類（資料請求/お問い合わせ）

カスタムディメンションの設定手順

ステップ1：イベントパラメータを送信

まず、GTMやgtagでイベント送信時にパラメータを追加します： 

gtag('event', 'generate_lead', {
  'form_type': '資料請求',
  'value': 100000
});

ステップ2：GA4でカスタムディメンションとして登録

  1. 管理画面から「カスタム定義」→「カスタムディメンションを作成」をクリック
  2. ディメンション名：「フォーム種類」（レポートに表示される名前）
  3. 範囲：「イベント」を選択
  4. イベントパラメータ：form_type（送信したパラメータ名）
  5. 「保存」をクリック

設定後24時間程度でレポートに反映されます。

BtoB向け推奨カスタムディメンション

ディメンション名 パラメータ名 用途
フォーム種類 form_type どのフォームから問い合わせがあったか
ログイン状態 login_status ログインユーザーの行動分析
ダウンロード資料名 document_name どの資料が人気か
業種 industry 業種別の分析
企業規模 company_size 企業規模別の分析

これらのディメンションを設定することで、BtoBマーケティング施策の効果をより詳細に分析できます。

カスタム指標の設定

カスタム指標は数値データです。例：

  • 動画視聴率
  • スクロール深度
  • フォーム入力時間

設定方法はカスタムディメンションと同様ですが、「範囲」で「イベント」または「ユーザー」を選び、数値型のパラメータを指定します。

上限に注意

カスタムディメンションは最大50個、カスタム指標は最大50個まで設定可能です（無料版GA4の場合）。本当に必要なものだけを厳選して設定しましょう。

GTM（Googleタグマネージャー）との連携設定

GTMを使うと、GA4の計測設定が格段に効率化されます。実務では必須のツールです。

GTMとは

Googleタグマネージャー（GTM）は、Webサイトのタグを一元管理できる無料ツールです。メリット：

  • エンジニアに依頼せずタグを追加・変更できる
  • 複数のタグを一括管理
  • 条件に応じたタグ発火が可能
  • プレビューモードでテスト可能
  • バージョン管理で過去に戻せる

GTMの初期設定

1. GTMアカウントの作成

  1. Googleタグマネージャーにアクセス
  2. 「アカウントを作成」をクリック
  3. アカウント名と国を設定
  4. コンテナ名とターゲットプラットフォーム（ウェブ）を設定
  5. 利用規約に同意

2. GTMコードの設置

表示されるGTMコードを、全ページに設置します：

  • <head>内に1つ目のコード
  • <body>直後に2つ目のコード

この作業は一度だけ。以降、タグの追加・変更はGTM管理画面から行えます。

GTMでGA4タグを設定する手順

ステップ1：GA4設定タグの作成

  1. GTM管理画面で「タグ」→「新規」をクリック
  2. タグ名：「GA4 - 設定」
  3. タグタイプ：「Googleタグ」を選択
  4. タグID：GA4の測定ID（G-XXXXXXXXXX）を入力
  5. トリガー：「All Pages」（すべてのページ）を選択
  6. 「保存」をクリック

ステップ2：イベントタグの作成例

例として、お問い合わせフォーム送信を計測するイベントタグを作成します：

  1. 「タグ」→「新規」をクリック
  2. タグ名：「GA4 - イベント - フォーム送信」
  3. タグタイプ：「Googleアナリティクス：GA4イベント」
  4. 設定タグ：先ほど作成した「GA4 - 設定」を選択
  5. イベント名：generate_lead
  6. イベントパラメータ（オプション）：
    • パラメータ名：form_type
    • 値：お問い合わせ
  7. トリガー：フォーム送信完了ページの閲覧（後述）
  8. 「保存」をクリック

ステップ3：トリガーの設定

「フォーム送信完了ページの閲覧」トリガーを作成：

  1. 「トリガー」→「新規」をクリック
  2. トリガー名：「ページビュー - サンクスページ」
  3. トリガータイプ：「ページビュー」
  4. 条件：「Page URL」「含む」「/thanks/」
  5. 「保存」をクリック

ステップ4：公開前のプレビュー

  1. GTM画面右上の「プレビュー」をクリック
  2. サイトのURLを入力して「Connect」
  3. 実際にフォーム送信して、タグが発火するか確認
  4. GA4の「DebugView」でもイベントを確認

ステップ5：公開

  1. 問題なければGTM画面右上の「公開」をクリック
  2. バージョン名と説明を入力
  3. 「公開」をクリック

GTM活用の効果

GTMを導入することで、リード獲得施策の効果測定が驚くほど簡単になります。新しいLPを公開した際も、エンジニアに依頼せずマーケター自身でタグを設定できるため、スピーディーにPDCAを回せます。

便利な変数の設定

GTMには「変数」機能があり、動的な値を取得できます。便利な変数：

組み込み変数

  • Page URL：現在のページURL
  • Page Path：URLのパス部分
  • Referrer：参照元URL
  • Click URL：クリックされたリンクのURL
  • Click Text：クリックされた要素のテキスト

これらは「変数」→「設定」から有効化できます。

カスタムJavaScript変数

JavaScriptで独自の変数を作成することも可能です。例えば、ログイン状態を取得する変数： 

function() {
  return window.isLoggedIn ? 'ログイン済み' : '未ログイン';
}

この変数をイベントパラメータとして送信すれば、ログイン状態別の分析が可能になります。

BigQueryエクスポート設定（上級者向け）

GA4の大きな特徴の一つが、BigQueryへの無料エクスポートです。生データを取得できるため、高度な分析が可能になります。

BigQueryエクスポートのメリット

  • GA4の制限（サンプリングなど）を受けない
  • SQLで自由にデータ分析
  • 他のデータ（CRM、MAツールなど）と結合可能
  • 長期間のデータ保持
  • BIツール（Looker Studio、Tableauなど）で可視化

BigQueryエクスポートの設定手順

ステップ1：Google Cloudプロジェクトの作成

  1. Google Cloud Consoleにアクセス
  2. 新しいプロジェクトを作成
  3. プロジェクト名を入力（例：「GA4-BigQuery」）

ステップ2：BigQueryの有効化

  1. Google Cloud Consoleで「BigQuery」を検索
  2. BigQuery APIを有効化

ステップ3：GA4からエクスポート設定

  1. GA4管理画面の「BigQueryのリンク」を選択
  2. 「リンク」をクリック
  3. Google Cloudプロジェクトを選択
  4. データの場所を選択（日本の場合：「asia-northeast1（東京）」または「asia-northeast2（大阪）」）
  5. エクスポート頻度を選択：
    • 日次：毎日1回エクスポート（推奨）
    • ストリーミング：リアルタイムでエクスポート（追加料金あり）
  6. 「送信」をクリック

設定後、24時間以内にデータのエクスポートが開始されます。

BigQueryでのデータ確認

BigQueryにエクスポートされたデータは、以下の構造で保存されます：

  • プロジェクト名
  • └ データセット名：analytics_XXXXXXXXX
  • 　└ テーブル名：events_YYYYMMDD（日付ごとのテーブル）

SQLで簡単にクエリを実行できます。例えば、昨日のページビュー数を取得： 

SELECT
  COUNT(*) AS pageviews
FROM
  `プロジェクトID.analytics_XXXXXXXXX.events_*`
WHERE
  event_name = 'page_view'
  AND _TABLE_SUFFIX = FORMAT_DATE('%Y%m%d', DATE_SUB(CURRENT_DATE(), INTERVAL 1 DAY))

BigQueryの料金に注意

BigQueryは基本無料ですが、以下の料金が発生する可能性があります：

  • ストレージ：月10GBまで無料、以降は$0.02/GB
  • クエリ：月1TBまで無料、以降は$5/TB

通常の中小企業サイトなら、無料枠内で十分収まります。

BigQueryを活用した高度な分析例

BigQueryを使うことで、以下のような高度な分析が可能になります：

  • コンバージョンに至ったユーザーの行動パターン分析
  • チャネルごとのLTV（顧客生涯価値）算出
  • CRMデータと結合した受注分析
  • セッション間隔の分析
  • カスタムアトリビューションモデルの構築

BtoBマーケティングの高度化には、BigQueryを活用したデータ分析が不可欠です。

よくあるトラブルと解決方法

GA4の設定でよくあるトラブルと、その解決方法をまとめました。

トラブル1：データが計測されない

確認ポイント

  • タグが正しく設置されているか（ブラウザの開発者ツールで確認）
  • 測定IDが正しいか（G-XXXXXXXXXXの形式）
  • GTMを使用している場合、公開されているか
  • 広告ブロッカーが有効になっていないか
  • 社内IPアドレスからのアクセスを除外していないか

解決方法

  1. DebugViewでリアルタイムのイベント発火を確認
  2. GTMのプレビューモードで動作確認
  3. ブラウザの拡張機能「Tag Assistant」で診断

トラブル2：イベントが重複して計測される

原因

  • 直接HTMLとGTMで両方タグを設置している
  • GTMで同じイベントを複数のタグから送信している
  • SPAでページ遷移時に複数回発火している

解決方法

  • タグは1種類の方法（GTM推奨）で設置
  • GTMのトリガー条件を見直す
  • イベントの発火回数を制限する設定を追加

トラブル3：コンバージョンが正しくカウントされない

確認ポイント

  • イベントが正しく発火しているか（DebugViewで確認）
  • イベント名が正確か（スペルミスなど）
  • コンバージョンとしてマークされているか

解決方法

  1. 「イベント」レポートで該当イベントの発火を確認
  2. イベント名を確認し、コンバージョン設定と一致させる
  3. 設定後24時間は反映に時間がかかることを考慮

トラブル4：レポートにデータが表示されない

原因

  • カスタムディメンション・指標が反映されていない（24時間必要）
  • データの閾値（しきい値）制限に引っかかっている
  • フィルターで除外されている

解決方法

  • カスタム定義は設定後24時間待つ
  • データの閾値については、Googleシグナルを無効化すると表示されることも
  • フィルター設定を確認

トラブル5：参照元が「direct / (none)」になる

原因

  • HTTPSからHTTPへの遷移（リファラが送信されない）
  • リダイレクトが挟まっている
  • アプリからの遷移
  • QRコードからのアクセス

解決方法

  • URLにUTMパラメータを付与
  • 参照元除外リストを設定（自社ドメイン、決済サイトなど）
  • 全ページHTTPS化

トラブル対処の基本

GA4のトラブル対処の基本は、「DebugView」と「リアルタイムレポート」での確認です。これらを活用すれば、ほとんどの問題は原因を特定できます。それでも解決しない場合は、プロに相談することをおすすめします。

まとめ

GA4の設定は、一見複雑に見えますが、本記事の手順に従えば必ず完了できます。重要なポイントをおさらいしましょう。

GA4設定の重要ポイント

最低限必須の設定

  • GA4プロパティの作成とタグ設置
  • データストリームの設定
  • 拡張計測機能の有効化
  • 主要コンバージョンの設定

推奨設定

  • GTMでのタグ管理
  • カスタムイベントの計測
  • カスタムディメンション・指標の設定
  • Googleシグナルの有効化
  • データ保持期間の延長

上級者向け

  • BigQueryエクスポート
  • クロスドメイントラッキング
  • User IDの実装
  • eコマース計測（該当する場合）

GA4は、従来のUAと比べて設定の自由度が高く、より詳細なユーザー行動を計測できます。特にBtoBマーケティングでは、リードの質や成約率の向上に直結するデータが取得できるため、適切な設定が不可欠です。

本記事で解説した設定を実施すれば、Webサイトからのリード獲得やLPの効果測定が正確に行えるようになります。まずは基本設定から始め、徐々に高度な計測を追加していきましょう。

GA4の活用は、データドリブンなマーケティングの第一歩です。正しく設定し、データを活用することで、オウンドメディア運用やマーケティング施策全体の効果を最大化できます。

設定後のアクション

GA4を設定したら、定期的にレポートを確認する習慣をつけましょう。週次・月次でのレポート確認、四半期ごとの詳細分析など、PDCAサイクルを回すことが成果につながります。
